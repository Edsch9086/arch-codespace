Atomic Symbols & Organizational Patterns

Atomic Symbols - Naming Convention:

Each distribution has a unique two-letter atomic symbol derived from its name:

  • Gn = GNU (founding element)
  • Lx = Linux (kernel element)
  • First + Second Letter Pattern: Most distributions use their first and second letters (Db=Debian, Ub=Ubuntu, Ar=Arch, Gt=Gentoo)
  • Special Cases: Some use first letter + distinctive letter (Tl=Tails, Rp=Raspberry Pi OS, Os=openSUSE, Zr=Zorin)

Organizational Patterns

Periods (Rows) - Maturity & Specialization Levels:

  • Period 1: Foundational - GNU and Linux kernel (the base elements)
  • Period 2: Early Derivatives - First major distribution families established
  • Period 3: Modern Stable - Well-established, widely-adopted distributions
  • Period 4: Specialized/Rolling - Specialized editions and rolling releases
  • Period 5: Lightweight/Alternative - Lightweight variants and niche purposes
  • Period 6: Enterprise/Specialized - Enterprise-grade and specialized deployments
  • Period 7: Niche/Experimental - Highly specialized or experimental distributions

Groups (Columns) - Distribution Families & Characteristics:

  • Group 1: GNU/Linux - The foundational elements (only Period 1)
  • Group 2: Debian Family - apt/dpkg systems (atomic numbers 1-19)
  • Group 3: Ubuntu Variants - Ubuntu-based desktop distributions (within apt range)
  • Group 4: SUSE Family - YaST-based enterprise distributions (within rpm range)
  • Group 5: Red Hat Family - RPM-based, RHEL derivatives (atomic numbers 30-39)
  • Group 6: Arch Family - Rolling release, pacman package manager (atomic numbers 20-29)
  • Group 7: Gentoo Family - Source-based, emerge package manager (atomic numbers 40-49)
  • Group 8: Standalone/Independent - Unique package managers or approaches (atomic numbers 50-70)
  • Group 9: Mobile/Embedded - Android, mobile, and embedded systems (highest numbers in each range)

Atomic Number Pattern:

  • Formula: Based on Package Manager + Architecture + Release Model + Specialization
  • Package Manager (Primary Characteristic):
    • apt/dpkg (Debian-based): 1-19
    • pacman (Arch-based): 20-29
    • rpm (Red Hat-based): 30-39
    • emerge (Gentoo-based): 40-49
    • Other/Independent: 50-70
  • Architecture Support (Secondary Modifier):
    • x86_64 only: base number
    • ARM support (RPi, Armbian, Arch ARM): +1 within range
    • Mobile platforms: +5 to +15 within range
  • Release Model (Tertiary Modifier):
    • Stable/LTS: base assignment
    • Rolling (Arch, Garuda, openSUSE Tumbleweed): sequential within package manager range
    • Specialized/Security-focused (Tails, Kali, Pentoo): lower numbers within range (more fundamental)
  • Specialization Level (Final Modifier):
    • Community distributions: standard numbering
    • Enterprise-focused (RHEL, SUSE Enterprise): higher end of rpm range (30-39)
    • Highly specialized (ChromeOS, WebOS, Android variants): shift within category range
  • Examples:
    • Debian (foundational apt distro): 3
    • Tails (security-specialized Debian): 7
    • Arch (foundational pacman distro): 20
    • Manjaro (user-friendly rolling Arch): 21
    • RHEL (foundational rpm distro): 6
    • Fedora (rolling rpm distro): 31
    • Gentoo (foundational emerge distro): 40
    • ChromeOS (Gentoo-based specialized): 41
